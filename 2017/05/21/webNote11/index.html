box 盒子

盒布局

box

看完flex之后，又过来把css3的盒布局复习一下
display:box || inline-box
用在基于webkit的chrome浏览器时，要display: -webkit-box;
gecko内核的浏览器 display: -moz-box;

给容器container一个display: box;
内部的box元素就会改变原有的文档流动方式，使用盒布局默认的文档流动方式。

元素的布局方向 – box-orient

box-orient属性:

  • horizontal//水平线从左到右
  • vertical//垂直线从上到下
  • inline-axis(默认值)//表示盒子元素沿着内联轴编排它的子元素，表现为横向编排
  • block-axis//表示元素沿着块轴编排它的子元素，表现为垂直编排
  • inherit//继承父元素

元素的布局顺序 box-derection

box-derection属性：

  • normal(默认值)// godv
  • reverse//反向排列 vdog
  • inherit//继承父元素

调整元素位置 box-ordinal-group

box-ordinal-group属性：

  • integer //自然整数 越小越左 越大越右

弹性空间分配 box-flex

设置值 子元素宽度弹性变化

元素的对齐方式 box-pack box-align

box-pack为水平方向对齐
box-align为垂直方向对齐

两者共有属性：

  • start// 水平靠左 垂直靠顶
  • end// 水平靠右 垂直靠底
  • center//水平居中 垂直居中

box-pack属性：

  • justify//所有子元素散开显示 第一个跟最后一个元素分别紧靠左右 中间空间平均分配

box-align属性：

  • baseline//所有的子元素沿着基线显示
  • stretch(默认值)//如果子元素没设高度，会变的跟父元素一样。

box-shadow

阴影 其中inset属性是往内设置阴影
box-shadow: none||inset 水平偏移(必须) 垂直偏移(必须) 模糊度(可选) 阴影大小(可选) color

box-sizing (important)

IE 设置width和height 包含了border和padding 可是标准的是不包含的
为此要写大量hack 满足不同浏览器需求
css3 就增加了box-sizing 自由定义是否包含border和padding

  • content-box(默认值)//都不包含
  • padding-box//包含padding 不包含border
  • border-box//都包含
  • inherit//继承

盒子溢出内容

overflow-x 和overflow-y

  • auto//溢出就出现滚动条
  • scroll//不管怎么样 都有滚动条
  • hidden//溢出就藏起来
  • visible//溢出就溢出 照样显示
    （新增属性)
  • no-conten//溢出 不显示内容
  • no-displau//溢出 不显示元素

That’s all
